# Markdown 财报章节切割

按以下 6 步流程操作，核心原则：**Python 做廉价工作（正则找行号），AI 只做判断（识别目录、歧义消解）**。

## 工具脚本

流程中使用 `scripts/split_report.py`，提供三个子命令：

| 命令 | 用途 |
|------|------|
| `python3 scripts/split_report.py find <文件> <标题> --exclude-before N` | 模糊正则匹配，返回 `行号:行内容` |
| `python3 scripts/split_report.py normalize <原始行>` | 清洗 TOC 行：去省略号、页码、噪声 |
| `python3 scripts/split_report.py split <文件> <输出目录>` | 按行号切割，stdin 输入 `行号\t章节名` |
| `python3 scripts/split_report.py financial-sub <文件> <输出目录>` | 发现并自动切割财务报告子章节（审计报告、合并报表等） |
| `python3 scripts/split_report.py financial-sub <文件> <输出目录> --stdin-split` | 同上，但自动接受排名第一的候选行号并执行切割 |

**预处理说明**：`split_report.py` 自动对输入文本执行 Unicode NFKC 归一化，将 CJK 兼容表意文字（如康熙部首变体 `⾦` U+2FA6）转换为标准字符（`金` U+91D1）。这解决了部分财报 PDF 转换引入的字符变体问题（常见于银行股报告）。无需手动处理。

**标题匹配排序规则**：`find_sub_sections()` 按以下优先级排序候选行：
1. `##` 前缀 + 数字编号（如 `## 一、审计报告`）— 最高优先级
2. `##` 前缀短标题（<30字符）
3. 数字编号短行（<30字符，可能是 TOC 条目）
4. `##` 前缀长标题
5. 短行无 `##` 前缀（<20字符）
6. 中长行（<50字符）
7. 长行（正文内容）

## 第 1 步：渐进式发现目录

每次读 50 行，从第 1 行开始，判断当前内容是否**目录区域**。

目录区域特征（满足越多越可能是）：
- 密集排列的多行条目，每行包含"章节标题 + 省略号/点线 + 页码"
- 出现"目录"字样作为附近标题
- 连续多行包含"第X节"或"第X章"且带有页码后缀

发现目录后，记录目录的**结束行号** `toc_end_line`（目录最后一行），后续搜索章节标题时用 `--exclude-before toc_end_line` 排除目录区域。

如果读到文件 1/3 处仍未找到目录，停下来报告用户确认。

## 第 2 步：提取章节标题

读取目录完整内容，识别顶级章节标题（非子章节、非前言）。输出清洗后的标题列表。

规则：
- 顶级章节通常以"第X节"或"第X章"开头
- 排除不含页码的纯文字行（可能是前言或说明）
- 用 `normalize` 命令清洗，或手动剔除省略号和页码

输出格式示例：
```
第一节 重要提示、目录和释义
第二节 公司简介和主要财务指标
...
第十节 财务报告
```

## 第 3 步：逐章节模糊匹配找行号

对每个章节标题，执行：

```bash
python3 scripts/split_report.py find <文件> "<标题>" --exclude-before <toc_end_line>
```

模糊正则在每个字符间插入了 `\s*`，能匹配 PDF 转换导致的字符间多余空格（如 "第 一 节" 仍能匹配 "第一节"）。

结果有三种情况：
- **1 个匹配** → 直接确认行号
- **0 个匹配** → 进入三级兜底流程（见下方）
- **多个匹配** → 进入第 4 步歧义消解

### 第 3.A 步：零匹配三级兜底

当 `find` 对某章节返回 0 个候选时，按以下顺序递进：

**Tier 1：脚本宽泛重试**

```
1. 缩短标题重试（去掉括号内容、只保留前半部分），重新 find
2. 用章节号模式重试：find "第X节"（利用顶级章节编号规律的强约束）
   → fuzzy_regex 的 \s* 已处理字符间空格和标点噪声
```

**Tier 2：脚本计算搜索范围**

若 Tier 1 仍无结果，利用相邻已定位章节计算搜索范围：

```
确定范围 [A, B]：
  A = 前一已定位章节的行号（或文件第 1 行）
  B = 后一已定位章节的行号（或文件最后一行）
```

**Tier 3：AI 模式搜索 + 确认**（详见附录 A）

```
1. 在 [A, B] 内用 find "第X节" 搜索（排除目录区域后）
   → 若有候选：AI 读每个候选行前后 ~10 行上下文，确认哪个是正式章节开始
   
2. 若仍无结果（标题跨行/OCR 错字）：
   → 脚本搜 [A, B] 内所有"标题候选行"：
     短行（len<50）、## 前缀行、编号行（^\d+[、.．]、^[一二三四五六七八九十]+[、.．]）
   → AI 读每个候选行前后 ~10 行上下文，按顺序判断哪个是目标章节

3. 极端情况（标题候选行也找不到）：
   → AI 读范围首尾内容确认边界，读范围中段判断内容归属，渐进收缩定位
```

## 第 4 步：AI 歧义消解

仅当某章节有多个候选行号时触发。

对每个候选行号 N，读文件 [N-10, N+10] 行（总共约 20 行），判断：
- 该行上下文是**正式章节开始**（前面是页眉/空白/上一章结束，后面紧跟章节正文），还是**目录条目**（周围都是带页码的条目）、**交叉引用**（正文中提及）、**子章节**（层级不对）？
- 确认唯一正确的行号

如果确有多个正式章节开始（不常见），取第一个（按出现顺序）。

## 第 5 步：切割文件

将所有确认的 `<行号>\t<章节名>` 通过 stdin 传给 split 命令：

```bash
printf "14\t第一节 重要提示、目录和释义\n194\t第二节 公司简介和主要财务指标\n..." | \
  python3 scripts/split_report.py split <文件> data/split/
```

输出文件命名：`{原文件名}_ch{序号}_{章节名}.md`

## 第 6 步：验证报告

切割完成后汇总表格：

| 章节 | 行号范围 | 输出文件 |
|------|---------|-----------|
| ... | ... | ... |

并检查：
- 最后一个文件是否涵盖到文件末尾
- 总行数是否与原文件接近（允许少量差异，因页眉/空白行）
- 抽查首尾文件的起止内容是否正确

## 第 7 步：财务报告二次切割（可选）

当切割出的最后一章为"财务报告"时，可进一步拆分为 6 个子部分：

| 子部分 | 搜索关键词 | 过滤词（排除误匹配） | 说明 |
|--------|-----------|---------------------|------|
| 审计报告 | `一、审计报告`、`审计报告` | `详见`、`内部控制审计报告`、`内控审计报告`、`审计报告文号` 等 | 过滤正文中的交叉引用和内部控制审计报告 |
| 合并资产负债表 | `合并资产负债表` | `项目注释` | 过滤项目注释中的提及 |
| 合并利润表 | `合并利润表` | `项目注释` | 过滤项目注释中的提及 |
| 合并现金流量表 | `合并现金流量表` | `项目注释` | 过滤项目注释中的提及 |
| 合并所有者权益变动表 | `合并所有者权益变动表`、`合并股东权益变动表` | `项目注释` | 过滤项目注释中的提及 |
| 合并财务报表附注 | `财务报表附注`、`财务报表注释` | `详见`、`以及相关` 等 | 过滤交叉引用；Tier 1 回退到`公司基本情况` |

**自动排除项**：所有母公司级别报表（母公司资产负债表、母公司利润表、母公司现金流量表、母公司所有者权益变动表、公司资产负债表等）。

### 7.1 执行自动切割

```bash
python3 scripts/split_report.py financial-sub <财务报告.md> <输出目录> --stdin-split
```

`--stdin-split` 模式对每个子章节选择排名第一的候选行号并执行切割。适用于大多数标准格式的 A 股财报。

输出文件命名：`{原文件名}_ch{序号}_{子章节名}.md`

### 7.2 AI 辅助歧义消解

当自动切割结果不理想时（如某子章节有多个候选或缺失），先不带 `--stdin-split` 运行：

```bash
python3 scripts/split_report.py financial-sub <财务报告.md> <输出目录>
```

这会打印所有子章节的候选匹配列表。

**标记说明**：
- `*` — 有多个候选，需 AI 歧义消解（读每个候选行前后 ~10 行上下文，判断哪个是正式章节开始）
- `?` — 零候选，需 AI 在搜索范围内定位（见下方 7.2.1）

确认所有行号后，通过 `split` 命令手动切割：

```bash
printf "行号\t章节名\n行号\t章节名\n..." | python3 scripts/split_report.py split <文件> <输出目录>
```

### 7.2.1 零候选兜底（三级递进）

当某子章节 `find_sub_sections()` 返回 0 个候选时，脚本会自动执行 Tier 1 宽泛重试。若仍无结果，输出 `! Search range` 供 AI 使用。

**脚本自动执行（Tier 1）**：

| 缺失章节 | 宽泛关键词（自动尝试） | 说明 |
|---------|----------------------|------|
| 审计报告 | — | "审计报告"已经是最宽泛的关键词 |
| 合并资产负债表 | `资产负债表` | 会同时命中母公司报表，由脚本自动过滤 |
| 合并利润表 | `利润表` | 同上 |
| 合并现金流量表 | `现金流量表` | 同上 |
| 合并所有者权益变动表 | `所有者权益变动表`、`股东权益变动表` | 同上 |
| 合并财务报表附注 | `公司基本情况` | 部分财报无"财务报表附注"标题，以此为兜底 |

**AI 介入（Tier 2-3）**：

当 Tier 1 宽泛关键词也返回 0 候选时，脚本输出搜索范围：

```
[合并资产负债表] (0 candidates):
  ! Search range: lines 45-200 (between 审计报告 and 合并利润表)
```

范围由相邻已定位章节的首行确定。

然后进入 **Tier 3：AI 自适应二分查找**（详见附录 B）：
1. 在 [A, B] 内二分查找：读中点 ~10 行，根据内容特征判断当前属于哪个章节
2. 各子章节内容高度可区分（科目名/措辞/结构），通常 10 行足够判断方向
3. 若落在空白/过渡段无法判断，自适应扩展读 20-30 行
4. 定位到行号后，补充到 split 列表中

### 7.3 验证

- 审计报告文件应以审计意见开头
- 合并资产负债表文件第一行应为"合并资产负债表"
- 合并报表文件内容不应含母公司字样
- 附注文件通常最大（占财务报告 90%+ 行数）

## 处理边界情况

- **第一个章节标题在目录前**（如年报中"第一节"作为标题出现在目录前面）：这种情况目录也属于第一章的内容，`find` 时不排除目录区域（`--exclude-before 0`），让 AI 判断哪个是真正的章节起始
- **目录是表格格式**（`| 第一节 | 标题... |`）：第 2 步提取标题时注意去除表格管道符和多余空格，提取纯标题文本
- **章节标题带 markdown 前缀**（`## 第一节 标题`）：模糊正则已支持可选 `#{1,3}` 前缀和表格管道符，无需手动处理
- **章节标题跨行**：PDF 转换可能导致标题折行，`fuzzy_find` 逐行匹配会漏掉。Tier 1 宽泛重试（缩短标题/章节号模式）通常能覆盖；若仍失败，进入 Tier 2-3 兜底流程
- **标题含特殊符号**：`re.escape` 自动转义，无需手动处理
- **摘要报告**（文件 < 1000 行，如年度报告摘要）：通常不含完整目录和财务报告子章节，章节数较少（3 节左右）。跳过目录发现步骤（第 1 步），直接使用 `find` 搜索 `第X节` 定位章节。`financial-sub` 对此类文件不适用
- **CJK 兼容表意文字**：部分 PDF 转换引入康熙部首等变体字符（如 `⾦` 替代 `金`、`⼀` 替代 `一`），多见于银行股报告。`split_report.py` 已通过 NFKC 归一化自动处理，`fuzzy_find`、`normalize_title` 等函数在匹配前统一转换

## Token 消耗基准

以下为 2026-05-15 对同一份财报（巨轮智能 2025 年报，12,694 行，430,162 字符）的对比测试结果：

| 指标 | 方法 1：AI 读全文 | 方法 2：本 Skill | 节省 |
|------|------:|------:|------:|
| 输入 token | 314,332 | 1,493 | 312,839 |
| 输出 token | 209 | 70 | 139 |
| **总计** | **314,541** | **1,563** | **312,978** |

- **节省率：99.5%**（方法 1 消耗是方法 2 的 201 倍）
- **切割效果：完全一致**（两种方法使用相同的行号和切割函数，输出 MD5 相同）
- **方法 2 的 token 来源**：渐进读取 100 行（704 tokens） + Python 命令输出（789 tokens） + AI 输出（70 tokens）
- **方法 1 的 token 来源**：全文读入上下文（314,332 tokens）占 99.9%，AI 分析输出仅占 0.1%

## 附录 A：顶级切割 Tier 3 模式搜索 + AI 确认

顶级章节具有强规律（`第X节` / `第X章`），利用此规律在搜索范围内定位。

```
给定：文件路径、搜索范围 [A, B]、目标章节号 X

1. find "第X节" --exclude-before A
   → fuzzy_regex 的 \s* 已处理字符间空格
   → 若有候选（通常 1-2 个）：AI 读每个候选行前后 ~10 行确认
   
2. 若仍无结果（标题跨行/OCR 错字）：
   → 搜 [A, B] 内所有"标题候选行"：
     · 短行（len < 50）
     · ## 前缀行
     · 编号行（^\d+[、.．]、^[一二三四五六七八九十]+[、.．]）
   → 通常 10-20 个候选，AI 按行号顺序读上下文辨认

3. 极端情况（全文 OCR 质量极差）：
   → 读 [A, B] 首尾附近内容确认边界
   → 读范围中段判断内容归属（与前/后章节的相似度），渐进收缩
```

Token 消耗：AI 只读候选行上下文（每处 ~10 行），Tier 1 命中时仅 2-3 个候选行。

## 附录 B：financial-sub Tier 3 自适应二分查找

财报各子章节内容高度可区分，利用内容特征在搜索范围内二分查找。

```
给定：文件路径、搜索范围 [A, B]、目标子章节

1. lo = A, hi = B
2. while lo < hi:
     mid = (lo + hi) // 2
     读 [mid-5, mid+5] 共 10 行
     判断 mid 位置属于：
       a. 目标章节之前的内容 → lo = mid + 1
       b. 目标章节本身 → 确认行号，返回
       c. 目标章节之后的内容 → hi = mid - 1
       d. 无法判断（空白/过渡段）→ 扩展读 20-30 行，直到能判断
3. 读 [lo-10, lo+10] 确认最终行号并验证
```

各子章节内容特征速查：

| 子章节 | 典型内容特征 |
|--------|-------------|
| 审计报告 | "我们认为"、"审计意见"、"关键审计事项"、"形成审计意见的基础" |
| 合并资产负债表 | "货币资金"、"应收账款"、"资产总计"、"负债和所有者权益" |
| 合并利润表 | "营业收入"、"营业成本"、"净利润"、"每股收益" |
| 合并现金流量表 | "经营活动产生的现金流量"、"投资活动"、"筹资活动" |
| 合并所有者权益变动表 | "股本"、"资本公积"、"盈余公积"、"未分配利润" |
| 合并财务报表附注 | "公司基本情况"、"重要会计政策"、"合并财务报表范围" |

每次迭代 10 行（~200 tokens），1000 行范围最多 10 次迭代 ≈ 2000 tokens。相比读整段（数万 tokens）节省 90%+。
