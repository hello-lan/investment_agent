# Markdown 财报章节切割

使用 `auto` 命令一键完成端到端切割。脚本内部自动完成 TOC 发现、标题提取、行号查找、规则消歧、文件切割。

**核心设计原则**：脚本完成 90% 的机械工作（单次 shell 调用），遇到不确定性时输出**实际文件上下文**供 LLM 精准决策，避免 LLM 反复用 `run_command` 试错。

## ⚠️ 输出目录约束

- **必须使用委派指令中指定的输出目录**，不得自行创建新目录或更改目录命名
- 当在 `orch-full-financial-analysis` 编排流程中使用时，输出目录约定为 `3_split/{股票代码}_{年份}/`
- 多文件批量切割时，每个文件创建对应的年份子目录，统一放在指定的输出根目录下

## 主流程

### 第 1 步：一键自动切割

```bash
python3 extensions/skills/split-financial-report/scripts/split_report.py auto <输入文件.md> <输出目录> [--financial-sub]
```

- `--financial-sub`：最后一章为"财务报告"时自动执行二次切割（审计报告、合并报表、附注等）

### 第 2 步：检查退出码

| exit | 含义 | LLM 需要做什么 |
|------|------|---------------|
| **0** | 全部成功 | 无需介入。跳转到验证步骤。 |
| **1** | 部分成功 | 脚本已切割能确认的章节。**仔细读脚本输出**——它已打印了 TOC 原始内容、未识别条目的行号和周围文本、搜索范围。LLM 根据这些信息确定正确行号后，用 `printf "行号\t章节名\n" \| python3 split_report.py split <文件> <输出>` 补切缺失章节。 |
| **2** | 无法切割 | 脚本无法识别文档结构。**读脚本输出的文件样本（前 100 行）**，确定文档格式后用 grep/find 手工定位章节，然后 pipe 给 split 命令。 |

**关键**：exit 1/2 时，脚本输出中已包含实际文件内容（带行号的原文），**不需要额外 `run_command` 去读文件**。直接看输出做决策即可。

### 第 3 步：验证

检查切割结果：
- 章节数是否与 TOC 一致
- 最后一个文件是否覆盖到文件末尾
- 抽查首尾文件内容是否正确

## LLM 介入场景与操作

### 场景 A：TOC 中有非标准章节号（exit 1）

脚本输出示例：
```
⚠ TOC has ~10 entries but only 9 parsed as chapters.
→ TOC region (✓=parsed, ?=unrecognized):
  ✓ [64] 第一节重要提示...
  ? [72] Part5 可持续报告.........62
  ✓ [74] 第六节重要事项...
```

**LLM 操作**：
1. 看 `?` 标记的行 — 这是脚本不认识的章节格式
2. 用 `grep -n "Part5\|可持续报告" <文件>` 在正文中定位该章节
3. 将行号补充到切割结果：
```bash
printf "2772\tPart5 可持续报告\n" | python3 split_report.py split <文件> <输出>
```
4. 如果章节名格式与已有不统一，统一命名后重新切割

### 场景 B：某章节在正文中定位失败（exit 1）

脚本会打印每个未定位章节的：
- 搜索范围（相邻已定位章节的行号区间）
- 范围中点周围的原文（约 25 行）
- 章节号模式在范围内的所有匹配行

**LLM 操作**：
1. 看打印的上下文，确认章节正文标题的确切行号
2. 运行补切命令

### 场景 C：完全无法识别文档格式（exit 2）

脚本打印文件前 100 行。

**LLM 操作**：
1. 阅读文件样本，理解文档的章节标题格式
2. 用 `grep -n` 找到所有章节正文标题的行号
3. 构建切割输入：
```bash
printf "行号\t章节名\n行号\t章节名\n..." | python3 split_report.py split <文件> <输出>
```

### 场景 D：financial-sub 有歧义候选（exit 1）

脚本对每个歧义子章节打印：
- Top 5 候选行号及其 `heading_score`（0=最强标题信号）
- 每个候选前后 ±3 行的原文上下文

**LLM 操作**：
1. 看 heading_score 和上下文，确认每个子章节的正确行号
2. 通过 stdin 传给 split：
```bash
printf "334\t审计报告\n565\t合并利润表\n..." | python3 split_report.py split <财务报告.md> <输出>
```

## 工具脚本参考

`split_report.py` 提供以下子命令：

| 命令 | 用途 |
|------|------|
| `auto <文件> <输出> [--financial-sub]` | **一键端到端切割**（推荐） |
| `find <文件> <标题> --exclude-before N` | 模糊正则匹配，返回 `行号:行内容` |
| `split <文件> <输出>` | 按行号切割，stdin 输入 `行号\t章节名` |
| `financial-sub <文件> <输出> [--stdin-split]` | 财务报告子章节发现+切割 |
| `normalize <原始行>` | 清洗 TOC 行：去省略号、页码、噪声 |

## 财务报告二次切割

`auto --financial-sub` 自动将财务报告章节拆分为 6 个子部分：

审计报告 → 合并资产负债表 → 合并利润表 → 合并现金流量表 → 合并所有者权益变动表 → 合并财务报表附注

脚本自动匹配标准 A 股格式（`一、审计报告`、`1、合并资产负债表`、`七、合并财务报表项目注释` 等），高置信度时自动采纳，有歧义时输出候选+上下文供 LLM 确认。

### 边界情况

- **零候选**：部分财报用非标准标题（如无"财务报表附注"标题）。脚本输出搜索范围，LLM 二分查找定位
- **审计报告多候选**：正文中大量引用"审计报告"。脚本的 heading_score 优先 `## 一、审计报告` 格式
- **母公司报表干扰**：脚本自动过滤母公司资产负债表/利润表/现金流量表

---

## 处理边界情况

- **CJK 兼容字符**（`⾦`→`金`、`⽬`→`目` 等）：NFKC 归一化自动处理
- **表格式 TOC**（`| 第一节 \| 标题... |`）：自动清洗管道符
- **第一节在目录前**：首章不排除 TOC 区域搜索
- **摘要报告**（< 1000 行）：TOC 检测失败时回退为 `## 第X节` 标题扫描
- **章节名含空格/特殊字符**：fuzzy_find 在每字符间插入 `\s*` 处理 PDF 转换噪声
