# A股财报水分识别与风险分析

## 分析方法论

本技能基于唐朝《手把手教你读财报（新准则升级版）》的系统化财报分析方法，核心思路：**以资产负债表为基石，三表联动交叉验证，识别财务异常信号并追溯证据链**。

分析顺序：先看审计意见和关键高管变动（排除硬伤）→ 资产负债表（资产质量、债务安全）→ 利润表（盈利质量）→ 现金流量表（造血能力）→ 三表交叉验证 → 综合定级。

## Token 管理策略

切割后的单章节文件大多 150-350 行（约 6,000-15,000 字符），完整读取不会导致上下文溢出。
但运行时压缩机制每 N 步会卸载旧 tool_result，因此**不可依赖"读过的内容留在上下文"**。

对策：
1. **每个文件只读一次** — 用 `cat` 完整读取 150-350 行的章节文件，不反复回头查
2. **边读边写 working_data.md** — 每读完一个文件，立即提取关键数据追加写入中间文件，写入命令的短返回不会被卸载
3. **附注用 grep 定位 + 读段落** — 附注文件 (~6700行) 用 grep 定位关键段落，再 sed 读该段落
4. **分析阶段只读 working_data.md** — 紧凑的结构化数据 (~2000字符)，一次 cat 全部载入，不触发卸载

## 工作流程

支持单年和多年两种模式。工作流程分四阶段：**年份检测** → **数据采集** → **分析** → **报告输出**。

### 阶段〇：年份检测（确定分析范围）

先列出切割目录，确定有哪些年份的数据：

```bash
ls -d {split_dir}/*/
```

输出示例：
- 单年：`2021/` → 单年分析模式
- 多年：`2021/ 2022/ 2023/` → 多年分析模式

中间文件路径：`data/reports/{code}/4_output/working_data.md`
输出目录需先用 `mkdir -p` 创建。

**先用一次 ls 匹配所有文件路径**（避免后续猜错文件名浪费步骤）：

```bash
ls {split_dir}/*审计报告* {split_dir}/*主要财务指标* {split_dir}/*重要事项* {split_dir}/*股东* {split_dir}/*合并资产负债表* {split_dir}/*合并利润表* {split_dir}/*合并现金流量表* {split_dir}/*合并财务报表附注*
```

从输出中确认每个章节文件的准确路径后，再开始逐章读取。

### 第一阶段：数据采集（按章节维度读取，每年同一章节连续读）

按以下顺序处理每一类章节文件。**有多年数据时，同一章节的多年文件连续读取、合并提取**，避免读完一年所有文件再读下一年（那会导致后期文件内容被卸载后前期的跨年对比数据丢失）。

采集前先创建 working_data.md 骨架：

```bash
cat > data/reports/{code}/4_output/working_data.md << 'EOF'
## {公司名} 财务数据提取（{year1}-{yearN}）

EOF
```

每提取完一类章节的数据，立即以表格形式追加写入 working_data.md。表格格式（多年模式）：

```markdown
### {章节名}
| 项目 | {year1} | {year2} | {year3} |
|------|---------|---------|---------|
```

单年模式去掉 year2/year3 列。

**章节 1: 审计报告**（每文件 ~246行 × N年，完整读取）

对每个年份依次 cat 审计报告文件，一次性提取所有年份的审计意见后写入表格：

```bash
# 多年模式：依次读取各年审计报告，提取后写入
cat {split_dir}/2021/...ch01_审计报告.md
cat {split_dir}/2022/...ch01_审计报告.md
cat {split_dir}/2023/...ch01_审计报告.md
```

提取项：审计意见类型、会计师事务所名称、签字会计师、审计年限（从聘任日期推算）。

**章节 2: ch02 主要财务指标**（每文件 ~184行 × N年，完整读取）

依次 cat 各年 ch02 文件。

提取项：营业收入、归母净利润、扣非净利润、经营CF净额、加权平均ROE、基本EPS、总资产、归母净资产。
每个 ch02 文件通常包含当年 + 前两年的"主要会计数据"表，可交叉验证。

**章节 3: ch06 重要事项**（每文件 ~320行 × N年，完整读取）

依次 cat 各年 ch06 文件。

提取项：关联交易金额及占比、对外担保情况、会计师事务所聘任信息（任期年限）、资金占用情况、承诺事项履行。

**章节 4: ch07 股东情况**（每文件 ~253行 × N年，完整读取）

依次 cat 各年 ch07 文件。

提取项：前十大股东名称及持股比例、控股股东持股比例及质押数量、冻结情况、实际控制人。

**章节 5: 合并资产负债表**（每文件 ~199行 × N年，完整读取）

依次 cat 各年合并资产负债表文件。

这是最重要的章节。提取所有科目的期末余额（多年模式下各年数据放入同一表格，便于跨年对比）。
重点关注科目：货币资金、应收票据（区分银票/商票）、应收账款、应收款项融资、预付款项、其他应收款、存货、
固定资产、在建工程、使用权资产、无形资产、商誉、长期待摊费用、
短期借款、应付票据、应付账款、合同负债、其他应付款、一年内到期的非流动负债、长期借款、应付债券、租赁负债、
资产总计、负债合计、归母所有者权益、少数股东权益。

**注意区分合并报表和母公司报表，只提取合并报表数据。**
多年模式下，资产负债表同时提供期末和期初数，可相互验证相邻年份数据一致性。

**章节 6: 合并利润表**（每文件 ~186行 × N年，完整读取）

依次 cat 各年合并利润表文件。

提取项：营业收入、营业成本、税金及附加、销售费用、管理费用、研发费用、财务费用（其中利息收入/利息费用）、
其他收益、投资收益、信用减值损失、资产减值损失、资产处置收益、营业利润、营业外收入/支出、利润总额、
所得税费用、净利润、归母净利润、扣非净利润。**只提取合并报表数据。**

**章节 7: 合并现金流量表**（每文件 ~162行 × N年，完整读取）

依次 cat 各年合并现金流量表文件。

提取项：经营活动CF净额、投资活动CF净额、筹资活动CF净额、
销售商品提供劳务收到的现金、购买商品接受劳务支付的现金、购建固定资产支付现金、
取得借款收到的现金、偿还债务支付的现金、分配股利或偿付利息支付的现金。
根据三类现金流的正负号确定各年"现金流量肖像"。

**章节 8: 附注**（1 步）

用技能内置脚本一次性提取附注所有关键段落：

```bash
bash extensions/skills/a-share-financial-forensic/scripts/extract_notes.sh {附注文件路径} data/reports/{code}/4_output/working_data.md
```

脚本自动 grep 定位 + sed 提取：货币资金(含受限)、应收票据+应收账款(含账龄/坏账)、存货(含跌价)、固定资产折旧+在建工程、短期/长期借款。结果直接追加到 working_data.md。

多年模式只需对最新一年执行一次。

### 第二阶段：数据分析

```bash
cat data/reports/{code}/4_output/working_data.md
```

此时所有年份的关键数据已在上下文中。**后续分析全部通过 LLM 推理完成，禁止再调用工具读取参考清单文件（技能正文已含完整步骤和38条规则），仅写入报告时使用 run_command。**

#### ⚠️ 非刚性分析原则

**本节的逐项检查清单是分析的下限，不是上限。** 38条规则覆盖了最常见的财务疑点，但实际分析中可能发现规则表之外的重要信号。遵循以下原则：

1. **先过规则表，再跳出规则表** — 逐条对照38条规则后，通读一遍 working_data.md 中的完整数据，问自己：有没有任何数据异常即使不在规则表中也值得投资者关注？
2. **趋势 > 绝对值** — 即使某个指标未触发阈值，但其年度变化趋势恶化（如质押比例连续攀升、账龄结构逐年迁移），必须作为关注项纳入报告
3. **跨指标联动 > 单指标孤立判断** — 经典造假信号往往不在单一规则中（如"毛利率↑ + 存货周转↑"组合），发现此类联动信号必须写入报告
4. **重大发现不拘泥于现有框架** — 若发现规则表未覆盖但实质重要的信号（如：大股东质押比例三年攀升逼近警戒线、受限资金构成突变、被收购标的业绩承诺期内精准达标后变脸），应在报告中新增独立小节详述

按以下顺序分析：

#### A. 基础审查（硬伤检查）

逐项对照规则：R019（审计意见非标）、R020（3年内换所>1次）、关键高管变动、R021（质押>50%）、R030（司法冻结）、R022（关联交易>30%）。
多年模式额外关注：事务所变更的年份分布。

**大股东质押分析必须覆盖以下全部项**（缺一不可）：
- 控股股东/实控人姓名及持股比例、质押股数、**质押占其持股比例各年趋势**（不只判是否>50%，更要看三年变化方向）
- 若质押比例连续攀升（如每年+5%以上），即使未触及50%也必须列入关注/风险项
- 质押资金用途是否披露、是否存在冻结记录（R030）
- 是否存在非经营性资金占用和违规担保
- 其他大股东（>5%）质押情况

#### B. 资产负债表分析

**货币资金**（R010/R011/R012）：多年模式下计算各年利息收入/货币资金平均余额，判断存贷双高是否持续存在。

**货币资金分析必须覆盖以下全部项**（缺一不可）：
- 货币资金总额各年值
- **受限资金细分**：定期存款及利息金额、各类保证金金额（银承/信用证/远期结售汇等）、其他受限类型
- 受限资金/货币资金比例各年趋势（R012要求：>30%触发High）
- 利息收入/货币资金均值各年比例（R011要求：<0.5%触发High）
- 有息负债总额各年值、利息费用各年值
- "存贷双高"综合判断（同时持有大额货币资金和大额有息负债的原因分析）

**应收款项**（R001/R002/R003/R004）：多年模式下计算应收增速-营收增速的连续差值，是最核心的跨年指标之一。
表格格式下直接对比各年应收/营收比例变化。

**应收款项分析必须覆盖以下全部项**（缺一不可）：
- 应收账款净额各年值和同比增速
- 应收/营收比例各年变化
- **账龄结构趋势**：1年以内占比 + **1年以上应收金额和增长率**（R003要求：>50%触发High）
- 坏账计提比例各年变化及原因（如信用保险组合影响）
- 坏账是否出现净转回（转回金额占净利润比例）

**存货**（R005/R006/R007/R036）：多年模式下重点关注"存货周转天数上升 + 毛利率上升"组合是否在多年间持续。

**存货分析必须覆盖以下全部项**（缺一不可）：
- 存货账面价值各年值和同比增速
- 存货/营业成本各年比率
- 存货跌价准备各年值和计提比例（R006/R036）
- 存货构成明细：原材料、在产品、库存商品、发出商品各年值及跌价比例
- **毛利率+存货周转联动分析**：毛利率上升 + 存货周转天数延长 = 超量生产摊薄固定成本的经典信号，无论是否触及R005/R007阈值，均须纳入关注项

**长期资产**（R008/R009/R024/R025）：多年模式下跟踪在建工程转固进度、商誉减值测试的年度一致性。

**有息负债**：计算各年有息负债/总资产趋势、现金债务比变化方向。

**快速阅读四指标**：生产资产/总资产、应收/总资产、现金/有息负债、非主业资产/总资产。多年模式下观察各指标的变化趋势。

#### C. 利润表分析

**收入质量**：多年模式下计算营收复合增速 vs 利润复合增速、各年关联交易占比变化。

**毛利率**（R015/R032）：多年模式下列出各年毛利率，直接观察波动幅度和趋势方向。

**费用结构**（R018）：多年模式下比较各年费用率变化，识别异常跳变年份。

**利润质量**（R016/R017/R037/R038）：多年模式下观察非经常性损益占比的年度变化、扣非/净利润差距的扩大趋势。

#### D. 现金流量表分析

**核心指标**（R013/R014/R035）：多年模式下列出各年经营CF/净利润比值和销售收现比，直接判断是否连续不达标。

**现金流量肖像**：必须逐年显式标出三类现金流的正负符号和对应类型名（妖精型/老母鸡型/蛮牛型/奶牛型/大出血型/赌徒型/混吃等死型/骗吃骗喝型），观察肖像类型的年度变化（如蛮牛型→奶牛型的进化路径，或奶牛型→蛮牛型的恶化路径）。

**自由现金流**：各年经营CF - 购建固定资产支付现金，多年模式下观察是否持续为负。

#### E. 三表联动交叉验证多年模式下的核心交叉点：
- 营收增长 vs 应收增长 vs 经营CF（三线是否同向？多年背离→高风险）
- 净利润增长 vs 经营CF增长 vs 存货增长
- 毛利率趋势 vs 存货周转趋势（同升→超量生产操纵信号）
- 货币资金充裕 + 持续大额有息负债 → 存贷双高

#### F. 多年趋势综合判断（仅多年模式）

当有 ≥2 年数据时，额外评估：

1. **恶化趋势识别**：哪些指标在逐年恶化？（如应收周转连续下降、毛利率连续下滑、经营CF/净利润比值连续走低）
2. **改善趋势识别**：哪些指标在逐年改善？改善是否有现金流支撑？
3. **拐点识别**：是否有指标在某年突然跳变？（如毛利率突然大幅下降、费用率突然上升）
4. **一致性检查**：各年审计意见、事务所、会计政策是否连贯？变更发生在哪一年？

#### G. 综合定级

回答三个根本问题：
1. **利润是否为真？** — 多年经营现金流是否持续验证利润？
2. **利润是否可持续？** — 利润来源是主业还是非经常项目？趋势如何？
3. **维持利润是否需要大量资本投入？** — 多年资本开支是否持续侵蚀自由现金流？

杜邦分析（多年模式下各年分别计算，对比驱动因素变化）：
```
ROE = 净利润率 × 总资产周转率 × 杠杆系数
    = (净利润/营业收入) × (营业收入/总资产) × (总资产/净资产)
```
关注 ROE 的驱动因素在多年间的变化：利润率提升（可持续）还是加杠杆（不可持续）？

风险评级：
- **极高风险**：存在审计非标、资金链断裂风险、明显造假线索
- **高风险**：多项财务指标异常且无合理解释
- **存疑**：个别指标异常，需持续跟踪
- **暂未发现重大财务疑点**：各项检查通过

### 第三阶段：报告输出

使用 `references/report-template.md` 中的统一模板。多年模式下报告额外包含：

- 多年财务数据汇总表（营收/净利润/经营CF/ROE/毛利率等核心指标的年度对比表）
- 趋势分析小节（恶化/改善/拐点识别）
- 各年现金流量肖像变化说明

报告章节：
1. **结论摘要** — 总体评级 + 多年趋势判断 + 主要驱动因素
2. **数据质量与加工说明** — 已识别/缺失字段、口径说明（明确标注合并报表口径还是母公司口径）、覆盖年份
3. **硬伤红灯检查** — 审计意见、事务所变更（含年度分布）、治理异常
4. **利润表疑点** — 事实数据 + 规则依据（引用编号）+ 风险推断（多年并列对比）
5. **资产负债表疑点** — 同上
6. **现金流量表疑点** — 同上 + 各年现金流量肖像（逐标注类型名称）
7. **多年趋势分析**（仅多年模式）— 恶化/改善/拐点 + 核心指标年度趋势表

## 杜邦分析

ROE 拆解为三因子（详见 `references/metric-derivation.md`）：

```
ROE = 净利润率 × 总资产周转率 × 杠杆系数
    = (净利润/营业收入) × (营业收入/总资产) × (总资产/净资产)
```

分析时关注 ROE 变化的驱动因素：是利润率提升（可持续）还是加杠杆（不可持续）？

## 注意事项

1. **严格控制步骤数** — 子Agent 默认 max_steps=30。数据采集必须在 25 步内完成，留 5 步给读 working_data.md + 写报告。分析阶段全部用 LLM 推理，不消耗步骤。
2. **先检测年份再采集** — 第一步必须 ls 切割目录确定年份范围，再决定采集策略。
3. **多年模式按章节维度读取** — 同一章节的多年文件连续读取，提取为一张对比表后写入 working_data.md。不要读完一年所有文件再读下一年。
4. **用 cat 完整读取** — 切割后的单文件 150-350 行完全适合一次读取。不要用 grep 逐条搜索（附注除外）。
5. **附注用脚本 1 步完成** — `bash extensions/skills/a-share-financial-forensic/scripts/extract_notes.sh`。不得手动 grep/sed 附注文件。多年模式只读最新年附注。
6. **分析阶段禁止读参考文件** — 技能正文已包含完整分析步骤和38条规则，分析时直接用 LLM 推理，不消耗步骤数读取 checklist 文件。
7. **working_data.md 是分析阶段的唯一数据来源** — 确保所有关键数据都已写入后再进入第二阶段。多年模式下表格格式支持直接跨年对比，分析时无需回翻原始文件。
8. **遵守非刚性分析原则** — 38条规则是最低覆盖要求，不是上限。发现规则表外的重大信号必须补充进"额外重大发现"章节。趋势恶化即使未触及阈值也要标注。

12. 单项异常不得直接定性，必须三表交叉验证。
13. 关注行业差异、公司阶段、经济周期对指标的影响。
