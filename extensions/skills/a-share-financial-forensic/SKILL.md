# A股财报水分识别与风险分析

## 分析方法论

本技能基于唐朝《手把手教你读财报（新准则升级版）》的系统化财报分析方法，核心思路：**以资产负债表为基石，三表联动交叉验证，识别财务异常信号并追溯证据链**。

分析顺序：先看审计意见和关键高管变动（排除硬伤）→ 资产负债表（资产质量、债务安全）→ 利润表（盈利质量）→ 现金流量表（造血能力）→ 三表交叉验证 → 综合定级。

## 核心设计原则

财报 md 文件由 PDF 转换+切割而来，**不同公司、不同年份的格式和标题命名差异巨大**。因此：

- **脚本做机械工作**：文件发现、大小估算、结构化表格解析（三张报表）
- **LLM 做智能提取**：非结构化数据（附注、审计报告、治理章节）由 LLM 一次性读取原文提取
- **参考 split-financial-report 模式**：脚本输出清单+上下文，LLM 精准决策，避免反复试错

## 工作流程

支持单年和多年两种模式。分四阶段：**清单生成**（脚本1步）→ **数据加载**（每文件只读一次）→ **分析推理**（0次工具调用）→ **报告输出**。

### 阶段〇：数据清单生成（1步脚本）

使用 `collect_data.py` 生成 `data_manifest.md`——包含文件清单（大小+推荐读取方式）+ 三张报表结构化解析 + LLM读取指南。

```bash
python3 extensions/skills/a-share-financial-forensic/scripts/collect_data.py {split_dir} data/reports/{code}/4_output/ --code {code}
```

**输入**：`{split_dir}` 下的年份子目录（如 `2023/ 2024/ 2025/`），每年含切割后的章节 `.md` 文件。

**输出**：`data/reports/{code}/4_output/data_manifest.md`（~15-25KB），包含：
- **文件清单**：每个文件的路径、大小、包含的数据类型、推荐读取方式
- **LLM 读取指南**：核心约束、大文件分段策略、读取顺序建议
- **三张财务报表多年对比表**（已由脚本解析为结构化表格，不需再读原始文件）
- **文本章节位置索引**：每类文件在各年份的路径和大小

脚本 stderr 输出文件发现摘要（每年每类是否找到），用于确认数据完整性。

### 阶段一：数据加载（每文件只读一次）

**核心规则**：
1. **每个文件只读一次** — 读取前规划好该文件需要提取的所有数据点，一次读完全部提取
2. **读取前先估算大小** — 用 `wc -c <file>` 确认文件大小
3. **禁止反复读取** — 读完一个文件后禁止回头再读同一文件提取其他指标
4. **读完后纯推理** — 数据加载完成后进入阶段二，禁止再调用工具读文件

**第一步**：加载清单（1步）
```bash
cat data/reports/{code}/4_output/data_manifest.md
```
此时三张报表的结构化数据已在上下文中。文件清单中的路径信息指导后续读取。

**第 1.5 步**：校验脚本匹配结果（0-2步）

⚠️ **脚本的正则匹配可能因格式差异而出错**。在开始大量读取文件之前，必须快速校验匹配结果：

1. **检查 ⚠️ fallback 标记的文件** — data_manifest.md 的"文件匹配校验清单"中标记为 `⚠️ fallback` 的文件，用 `head -5` 快速确认文件类别是否匹配
2. **检查未匹配的大文件** — "🔍 未匹配文件"列表中 >100KB 的文件可能包含遗漏数据，用 `grep -l` 快速识别：
   ```bash
   grep -l '财务报表附注\|合并资产负债表\|审计报告' <未匹配的大文件>
   ```
3. **附注缺失时的 fallback** — 如果"合并财务报表附注"标注为"未找到"，data_manifest.md 的"⚠️ 附注缺失 Fallback"章节给出了按优先级排列的候选文件和确认命令。按提示排查即可
4. 若发现匹配错误：在后续加载步骤中使用正确的文件路径，忽略脚本的错误匹配

校验通过后进入实际数据加载。

**第二步**：按优先级读取数据源文件。data_manifest.md 中的"读取顺序建议"表格已给出每类文件的大小、读取方式和提取目标。遵循该表格执行即可。关键约束：

- **小文件（<50KB）**：`cat` 全文，一次性提取全部指标
- **大文件附注（>100KB）**：两步法 — 先 `grep -n` 一次性定位所有目标段落行号（1次工具调用），再 `sed -n` 一次性读取所有段落（1次工具调用）
- 同类型的多年文件可以合并为一次 cat（如 `cat ch02/2023.md ch02/2024.md ch02/2025.md`）

**步数预算**：

| 步骤 | 说明 | 工具调用 |
|------|------|----------|
| 1 | 执行 collect_data.py | 1 |
| 2 | cat data_manifest.md | 1 |
| 2.5 | 校验匹配结果（检查⚠️标记 + 未匹配文件） | 0-2 |
| 3 | cat 主要财务指标（3年合并） | 1 |
| 4 | cat 审计报告（3年合并） | 1 |
| 5 | cat 股东情况（3年合并） | 1 |
| 6 | cat 重要事项（3年合并） | 1 |
| 7-8 | 附注分段读取（grep 定位 + sed 读取） | 2 |
| 9-10 | 写入报告 | 1-2 |
| **合计** | | **9-13步**（均在15次预算内） |

完成上述读取后，38条规则所需的全部数据已在上下文中。进入阶段二前确认：三张报表 ✅ | 主要财务指标 ✅ | 审计报告 ✅ | 股东质押 ✅ | 关联交易 ✅ | 附注关键数据 ✅。

### 阶段二：分析推理（0次工具调用）

所有数据已在上下文中。**后续分析全部通过 LLM 推理完成，禁止再调用工具读取文件**（技能正文已含完整分析步骤和38条规则）。仅写入报告时使用 run_command。

#### ⚠️ 非刚性分析原则

**本节的逐项检查清单是分析的下限，不是上限。** 38条规则覆盖了最常见的财务疑点，但实际分析中可能发现规则表之外的重要信号。遵循以下原则：

1. **先过规则表，再跳出规则表** — 逐条对照38条规则后，通读一遍已加载的全部数据，问自己：有没有任何数据异常即使不在规则表中也值得投资者关注？
2. **趋势 > 绝对值** — 即使某个指标未触发阈值，但其年度变化趋势恶化（如质押比例连续攀升、账龄结构逐年迁移），必须作为关注项纳入报告
3. **跨指标联动 > 单指标孤立判断** — 经典造假信号往往不在单一规则中（如"毛利率↑ + 存货周转↑"组合），发现此类联动信号必须写入报告
4. **重大发现不拘泥于现有框架** — 若发现规则表未覆盖但实质重要的信号（如：大股东质押比例三年攀升逼近警戒线、受限资金构成突变、被收购标的业绩承诺期内精准达标后变脸），应在报告中新增独立小节详述

按以下顺序分析：

#### A. 基础审查（硬伤检查）

逐项对照规则：R019（审计意见非标）、R020（3年内换所>1次）、关键高管变动、R021（质押>50%）、R030（司法冻结）、R022（关联交易>30%）。
多年模式额外关注：事务所变更的年份分布。

**大股东质押分析必须覆盖以下全部项**（缺一不可）：
- 控股股东/实控人姓名及持股比例、质押股数、**质押占其持股比例各年趋势**（不只判是否>50%，更要看三年变化方向）
- 若质押比例连续攀升（如每年+5%以上），即使未触及50%也必须列入关注/风险项
- 质押资金用途是否披露、是否存在冻结记录（R030）
- 是否存在非经营性资金占用和违规担保
- 其他大股东（>5%）质押情况

#### B. 资产负债表分析

**货币资金**（R010/R011/R012）：多年模式下计算各年利息收入/货币资金平均余额，判断存贷双高是否持续存在。

**货币资金分析必须覆盖以下全部项**（缺一不可）：
- 货币资金总额各年值
- **受限资金细分**：定期存款及利息金额、各类保证金金额（银承/信用证/远期结售汇等）、其他受限类型
- 受限资金/货币资金比例各年趋势（R012要求：>30%触发High）
- 利息收入/货币资金均值各年比例（R011要求：<0.5%触发High）
- 有息负债总额各年值、利息费用各年值
- "存贷双高"综合判断（同时持有大额货币资金和大额有息负债的原因分析）

**应收款项**（R001/R002/R003/R004）：多年模式下计算应收增速-营收增速的连续差值，是最核心的跨年指标之一。
表格格式下直接对比各年应收/营收比例变化。

**应收款项分析必须覆盖以下全部项**（缺一不可）：
- 应收账款净额各年值和同比增速
- 应收/营收比例各年变化
- **账龄结构趋势**：1年以内占比 + **1年以上应收金额和增长率**（R003要求：>50%触发High）
- 坏账计提比例各年变化及原因（如信用保险组合影响）
- 坏账是否出现净转回（转回金额占净利润比例）

**存货**（R005/R006/R007/R036）：多年模式下重点关注"存货周转天数上升 + 毛利率上升"组合是否在多年间持续。

**存货分析必须覆盖以下全部项**（缺一不可）：
- 存货账面价值各年值和同比增速
- 存货/营业成本各年比率
- 存货跌价准备各年值和计提比例（R006/R036）
- 存货构成明细：原材料、在产品、库存商品、发出商品各年值及跌价比例
- **毛利率+存货周转联动分析**：毛利率上升 + 存货周转天数延长 = 超量生产摊薄固定成本的经典信号，无论是否触及R005/R007阈值，均须纳入关注项

**长期资产**（R008/R009/R024/R025）：多年模式下跟踪在建工程转固进度、商誉减值测试的年度一致性。

**有息负债**：计算各年有息负债/总资产趋势、现金债务比变化方向。

**快速阅读四指标**：生产资产/总资产、应收/总资产、现金/有息负债、非主业资产/总资产。多年模式下观察各指标的变化趋势。

#### C. 利润表分析

**收入质量**：多年模式下计算营收复合增速 vs 利润复合增速、各年关联交易占比变化。

**毛利率**（R015/R032）：多年模式下列出各年毛利率，直接观察波动幅度和趋势方向。

**费用结构**（R018）：多年模式下比较各年费用率变化，识别异常跳变年份。

**利润质量**（R016/R017/R037/R038）：多年模式下观察非经常性损益占比的年度变化、扣非/净利润差距的扩大趋势。

#### D. 现金流量表分析

**核心指标**（R013/R014/R035）：多年模式下列出各年经营CF/净利润比值和销售收现比，直接判断是否连续不达标。

**现金流量肖像**：必须逐年显式标出三类现金流的正负符号和对应类型名（妖精型/老母鸡型/蛮牛型/奶牛型/大出血型/赌徒型/混吃等死型/骗吃骗喝型），观察肖像类型的年度变化（如蛮牛型→奶牛型的进化路径，或奶牛型→蛮牛型的恶化路径）。

**自由现金流**：各年经营CF - 购建固定资产支付现金，多年模式下观察是否持续为负。

#### E. 三表联动交叉验证多年模式下的核心交叉点：
- 营收增长 vs 应收增长 vs 经营CF（三线是否同向？多年背离→高风险）
- 净利润增长 vs 经营CF增长 vs 存货增长
- 毛利率趋势 vs 存货周转趋势（同升→超量生产操纵信号）
- 货币资金充裕 + 持续大额有息负债 → 存贷双高

#### F. 多年趋势综合判断（仅多年模式）

当有 ≥2 年数据时，额外评估：

1. **恶化趋势识别**：哪些指标在逐年恶化？（如应收周转连续下降、毛利率连续下滑、经营CF/净利润比值连续走低）
2. **改善趋势识别**：哪些指标在逐年改善？改善是否有现金流支撑？
3. **拐点识别**：是否有指标在某年突然跳变？（如毛利率突然大幅下降、费用率突然上升）
4. **一致性检查**：各年审计意见、事务所、会计政策是否连贯？变更发生在哪一年？

#### G. 综合定级

回答三个根本问题：
1. **利润是否为真？** — 多年经营现金流是否持续验证利润？
2. **利润是否可持续？** — 利润来源是主业还是非经常项目？趋势如何？
3. **维持利润是否需要大量资本投入？** — 多年资本开支是否持续侵蚀自由现金流？

杜邦分析（多年模式下各年分别计算，对比驱动因素变化）：
```
ROE = 净利润率 × 总资产周转率 × 杠杆系数
    = (净利润/营业收入) × (营业收入/总资产) × (总资产/净资产)
```
关注 ROE 的驱动因素在多年间的变化：利润率提升（可持续）还是加杠杆（不可持续）？

风险评级：
- **极高风险**：存在审计非标、资金链断裂风险、明显造假线索
- **高风险**：多项财务指标异常且无合理解释
- **存疑**：个别指标异常，需持续跟踪
- **暂未发现重大财务疑点**：各项检查通过

### 阶段三：报告输出

使用 `references/report-template.md` 中的统一模板。多年模式下报告额外包含：

- 多年财务数据汇总表（营收/净利润/经营CF/ROE/毛利率等核心指标的年度对比表）
- 趋势分析小节（恶化/改善/拐点识别）
- 各年现金流量肖像变化说明

报告章节：
1. **结论摘要** — 总体评级 + 多年趋势判断 + 主要驱动因素
2. **数据质量与加工说明** — 已识别/缺失字段、口径说明（明确标注合并报表口径还是母公司口径）、覆盖年份
3. **硬伤红灯检查** — 审计意见、事务所变更（含年度分布）、治理异常
4. **利润表疑点** — 事实数据 + 规则依据（引用编号）+ 风险推断（多年并列对比）
5. **资产负债表疑点** — 同上
6. **现金流量表疑点** — 同上 + 各年现金流量肖像（逐标注类型名称）
7. **多年趋势分析**（仅多年模式）— 恶化/改善/拐点 + 核心指标年度趋势表
8. **三表联动交叉验证** — 营收↔应收↔经营CF、净利↔经营CF↔存货、毛利率↔存货周转、货币资金↔利息
9. **额外重大发现** — 38条规则表未覆盖但分析中发现的实质重要信号。若无可标注"无"。不限制内容范围。
10. **附注与董事会报告反证** — 关键发现、一致性判断
11. **三大前提判定** — 利润为真/可持续/资本投入（必须显式回答三个问题，缺一不可）
12. **风险分层清单** — 高/中/低风险信号（标注发现年份和触发规则编号）
13. **重点跟踪建议** — 按优先级 P0/P1/P2，每项含具体关注指标
14. **待补充数据** — 按优先级 P1/P2/P3

报告保存路径：
- 单年：`data/reports/{code}/4_output/{code}_{year}_财务排雷分析报告.md`
- 多年：`data/reports/{code}/4_output/{code}_{year_from}-{year_to}_财务排雷分析报告.md`

## 疑点规则库（38条）

分析过程中逐条对照以下规则，每条疑点必须落地到"事实数据 + 规则依据 + 风险推断"。

| 编号 | 科目 | 规则名 | 触发条件 | 严重度 |
|------|------|--------|---------|--------|
| R001 | 应收账款 | 应收增速长期高于营收增速 | 连续2年应收增幅-营收增幅>10% | High |
| R002 | 应收账款 | 应收/营收比例超30% | 制造业超过30%即需警惕 | High |
| R003 | 应收账款 | 长账龄应收账款大幅增长 | 1年以上应收回款增幅>50% | High |
| R004 | 应收账款 | 坏账计提政策变更 | 当年计提政策减少计提 | High |
| R005 | 存货 | 存货激增而营收未同步增长 | 存货增幅-营收增幅>20%连续2年 | High |
| R006 | 存货 | 存货跌价准备计提不足 | 计提比例显著低于行业 | Medium |
| R007 | 存货 | 存货周转天数显著延长 | 周转天数同比增加>30天 | Medium |
| R008 | 商誉 | 商誉占净资产比例超30% | 商誉/净资产>30% | Medium |

## 杜邦分析

ROE 拆解为三因子（详见 `references/metric-derivation.md`）：

```
ROE = 净利润率 × 总资产周转率 × 杠杆系数
    = (净利润/营业收入) × (营业收入/总资产) × (总资产/净资产)
```

分析时关注 ROE 变化的驱动因素：是利润率提升（可持续）还是加杠杆（不可持续）？

## 注意事项

1. **数据清单 1 步生成** — 执行 `collect_data.py` 生成 `data_manifest.md`（文件清单 + 结构化报表 + 读取指南），1步完成。
2. **每个文件只读一次** — 关键原则。读取前用 `wc -c` 确认大小，<50KB 全文 cat，>100KB 大文件（附注）用 grep定位+sed读取 两步策略。读完一个文件后禁止回头再读。
3. **数据加载后纯推理** — 所有数据加载完成后（阶段一），分析阶段（阶段二）禁止再调用工具读取任何文件。技能正文已包含完整分析步骤和38条规则。仅写入报告时使用 run_command。
4. **data_manifest.md 是导航地图** — 包含文件清单、大小、推荐读取方式、读取顺序建议、结构化财务报表。先 cat 它，再按它指引的顺序读取其他文件。
5. **遵守非刚性分析原则** — 38条规则是最低覆盖要求，不是上限。发现规则表外的重大信号必须补充进"额外重大发现"章节。趋势恶化即使未触及阈值也要标注。
6. **分析覆盖项必须完整** — 应收账款项必须含账龄趋势（1年以上增长率），货币资金项必须含受限资金细分，大股东项必须含质押比例年度趋势，存货项必须含毛利率+存货周转联动分析，现金流量肖像必须逐标注类型名称，三大前提判定必须显式回答全部三个问题。
7. **明确标注数据口径** — 必须使用合并报表数据（非母公司报表），若需引用母公司数据须显式标注。报告"数据质量"节标注覆盖年份、缺失字段、数据来源。
8. 关键结论必须有数据证据与规则依据（引用编号），不允许主观跳结论。
9. 单项异常不得直接定性，必须三表交叉验证。
10. 关注行业差异、公司阶段、经济周期对指标的影响。
11. 会计政策、合并范围变化影响可比性，必须单独说明。
12. 报告只做风险排查，不输出任何交易建议或收益承诺。
13. 存在疑问但证据不足时，标注"存疑"而非强行定性。
